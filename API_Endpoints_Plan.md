### Документ: План кінцевих точок API (Phase 1)

#### 1. Аутентифікація та Користувачі (`users` app)
`POST` `/api/v1/auth/login/` -> Вхід за номером телефону, видача JWT (access/refresh).

`POST` `/api/v1/auth/refresh/` -> Оновлення токена доступу.

`GET` `/api/v1/users/me/` -> Отримання профілю поточного користувача (роль, ПІБ, філії).

`GET` `/api/v1/users/teachers/` -> Список усіх викладачів (доступно для Admin).

#### 2. Управління структурою та навчальний процес (`core` app)
`GET` `/api/v1/branches/` -> Отримання списку філій, до яких прикріплений користувач.

`GET` `/api/v1/subjects/` -> Перелік навчальних дисциплін.

`GET` `/api/v1/students/` -> Список студентів (з фільтрацією за філією/групою).

`POST` `/api/v1/students/` -> Реєстрація нового студента в базі.

`GET` `/api/v1/groups/` -> Список активних навчальних груп.

`GET` `/api/v1/lessons/` -> Отримання розкладу занять.

`PATCH` `/api/v1/lessons/{id}/attendance/` -> Відмітка присутності студентів на конкретному занятті.
